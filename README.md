# BlockShot

> 블록을 쏘아 발판을 만들고, 스위치와 게이트를 활용해 포털까지 도달하는 2D 퍼즐 플랫포머 프로토타입입니다.

## 프로젝트 개요

`BlockShot`은 2020년 2학기에 개발한 Unity 기반 2D 플랫포머 게임 프로토타입입니다. 플레이어는 기본 이동과 점프만으로 도달하기 어려운 지형을, 직접 발사한 탄환으로 임시 블록을 생성해 돌파합니다. 각 스테이지는 클리어 시간에 따라 별 등급을 부여해 반복 플레이와 기록 단축을 유도합니다.

## 핵심 플레이

- 좌우 이동과 2단 점프로 지형을 돌파합니다.
- 방향키와 발사 키를 조합해 원하는 방향으로 탄환을 발사합니다.
- 탄환이 벽이나 바닥에 닿으면 일정 시간 유지되는 임시 블록이 생성됩니다.
- 생성한 블록을 발판으로 사용해 더 높은 위치나 먼 거리의 지형을 넘어갑니다.
- 스위치를 작동해 게이트를 열고 목표 지점인 포털에 도달합니다.
- 클리어 시간에 따라 1개부터 3개까지 별 등급을 획득합니다.

## 조작법

| 동작 | 입력 |
| --- | --- |
| 이동 | `Left Arrow`, `Right Arrow` |
| 점프 / 2단 점프 | `Space` |
| 블록 탄환 발사 | `S` + 방향키 |
| 재시작 / 스테이지 이동 | UI 버튼 |

## 주요 기능

### 블록 생성 슈팅

플레이어가 방향을 지정해 탄환을 발사하면, 탄환이 `Floor` 오브젝트와 충돌한 위치 기준으로 임시 블록이 생성됩니다. 생성된 블록은 일정 시간이 지나면 자동으로 사라져, 퍼즐 풀이와 이동 타이밍이 함께 요구됩니다.

### 제한 기반 쿨타임

탄환 발사 후에는 쿨타임이 적용되며, 플레이어 스프라이트 상태도 함께 변경됩니다. 이를 통해 현재 블록 생성 가능 여부를 시각적으로 구분할 수 있습니다.

### 스위치와 게이트

스테이지에는 스위치와 게이트 구조가 포함되어 있습니다. 플레이어가 스위치에 닿으면 게이트의 물리 충돌이 비활성화되어 막혀 있던 경로가 열립니다.

### 클리어 타임과 별 등급

각 스테이지는 별 3개와 별 2개 기준 시간을 별도로 갖고 있습니다. 포털에 도달하면 클리어 시간이 기록되고, 기준 시간에 따라 별 등급이 계산됩니다.

## 기술 스택

- Engine: Unity 2019.4.12f1
- Language: C#
- Rendering: Unity 2D Sprite
- UI: Unity UI, TextMeshPro 패키지 포함
- Platform: Windows 빌드 포함

## 프로젝트 구조

```text
BlockShot/
├── Project.3 Prototype/        # Unity 프로젝트
│   ├── Assets/
│   │   ├── Scenes/             # Start, StageSelect, Stage1~15, StageClear
│   │   ├── Prefabs/            # 스위치, 게이트, 별, 버튼 등 프리팹
│   │   ├── Character/          # 캐릭터 스프라이트 및 애니메이션 관련 스크립트
│   │   └── *.cs                # 플레이어, 탄환, 카메라, 타이머 등 핵심 스크립트
│   ├── Packages/
│   └── ProjectSettings/
├── C# 코드 모음/               # 주요 C# 스크립트 별도 정리본
├── 이미지 모음/                # 게임에 사용된 이미지 리소스
└── 실행 파일/                  # Windows 실행 빌드
```

## 주요 스크립트

| 스크립트 | 역할 |
| --- | --- |
| `CircleController.cs` | 플레이어 이동, 점프, 목표 지점 도달, 스위치 상호작용 |
| `BulletJenerator.cs` | 탄환 발사 입력, 발사 방향, 쿨타임, 캐릭터 상태 변경 |
| `BulletController.cs` | 탄환 이동, 충돌 시 임시 블록 생성 |
| `BlockDestroyer.cs` | 생성된 임시 블록의 자동 삭제 |
| `SwitchAndGate.cs` | 스위치 작동과 게이트 개폐 처리 |
| `TimerDirector.cs` | 스테이지 클리어 시간 측정 및 UI 표시 |
| `ClearDirector.cs` | 클리어 결과 화면, 별 등급 계산 |
| `TotalStarManager.cs` | 스테이지별 별 획득 정보 관리 |
| `CharacterSpriteChanger.cs` | 이동, 점프, 쿨타임 상태에 따른 캐릭터 스프라이트 변경 |

## 실행 방법

### 빌드 실행

Windows 환경에서 아래 파일을 실행합니다.

```text
실행 파일/BlockShot.exe
```

### Unity 프로젝트 열기

1. Unity Hub에서 Unity `2019.4.12f1` 버전을 설치합니다.
2. `Project.3 Prototype` 폴더를 Unity 프로젝트로 엽니다.
3. `Assets/Scenes/StartScene.unity` 또는 원하는 스테이지 씬을 실행합니다.

## 구현 포인트

- 단순한 이동 플랫포머에 블록 생성 슈팅을 결합해 퍼즐 풀이 요소를 추가했습니다.
- 스테이지별 제한 시간과 별 등급을 두어 클리어 이후에도 기록 단축 목표가 생기도록 구성했습니다.
- 스위치, 게이트, 임시 블록, 위험 블록, 포털 등 반복 사용 가능한 오브젝트를 프리팹 중심으로 구성했습니다.
- 총 15개의 스테이지를 제작해 핵심 메커니즘을 단계적으로 활용하도록 설계했습니다.

## 개선 가능점

- `GameObject.Find` 의존도를 줄이고 Inspector 참조 또는 매니저 초기화 구조로 안정성을 높일 수 있습니다.
- 탄환과 임시 블록 개수 계산 로직을 분리해 쿨타임 및 생성 제한을 더 명확하게 만들 수 있습니다.
- 클리어 기록과 별 획득 정보를 로컬 저장소에 저장하면 게임을 재실행해도 진행 상황을 유지할 수 있습니다.
- 캐릭터 이동을 `Rigidbody2D` 기반으로 통일하면 물리 충돌과 이동 판정을 더 안정적으로 관리할 수 있습니다.
